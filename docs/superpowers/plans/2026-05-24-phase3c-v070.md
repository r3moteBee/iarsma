# Phase 3c Implementation Plan (v0.7.0)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Push subscriptions replace polling for live updates. Action-log UI surfaces the tamper-evident audit trail. OpenInference export layer for observability tool integration.

**Architecture:** JMAP EventSource client invalidates Jotai cache atoms on state changes. Activity view queries the action-log store with filters and verifies hash-chain integrity. OI exporter maps entries to OpenInference spans.

**Tech Stack:** TypeScript, JMAP EventSource (RFC 8620 §7.3), React, Jotai, Vitest

**Spec:** `docs/superpowers/specs/2026-05-23-phase3-design.md` — Cut 3c (items 8-12)

### Task 1: Push Subscription (EventSource Client) — Item 8

**Files:**
- Create: `shell/src/runtime/push-subscription.ts`
- Create: `shell/src/runtime/__tests__/push-subscription.test.ts`
- Modify: `shell/src/App.tsx` (mount hook in SignedInView)

**Hook:** `usePushSubscription(session)` — manages EventSource lifecycle:
1. On mount: register push subscription via PushSubscription/set
2. Open EventSource to session's eventSourceUrl
3. On StateChange: map to cache invalidations (Email→threads/threadBodies/searchResults, Mailbox→mailboxes, Identity→identities)
4. On reconnect: issue Email/changes + Mailbox/changes with last state tokens for catch-up
5. Close on sign-out, close after 5min hidden tab, reopen on visible

**State-to-cache mapping:**
- `Email` → invalidate `threads`, `threadBodies`, `searchResults`
- `Mailbox` → invalidate `mailboxes`
- `Identity` → invalidate `identities`

### Task 4: Activity View (Action-Log UI) — Item 11

**Files:**
- Create: `shell/src/views/activity-view.tsx`
- Create: `shell/src/views/__tests__/activity-view.test.tsx`
- Modify: `shell/src/App.tsx` (enable Activity nav tab)
- Create: `shell/src/runtime/log-verifier.ts` (integrity verification with checkpoints)

**Layout:** Full-width paginated table (50/page):
- Columns: Timestamp | Actor | Action | Mode | Details
- Expandable row: full params JSON, provenance, hash-chain fields
- Filters: Actor (You/agent names/All), Action (multi-select), Mode (preview/commit/all), Time range

**Integrity verification:**
- `logVerifier.ts`: delta verification (since last checkpoint), full verification (periodic/on-demand)
- Checkpoint stored in IDB: { lastVerifiedHash, entryCount, timestamp }
- First load: full chain verification
- Subsequent: delta only
- Daily: full re-verification
- Failure: red banner with entry number
- Manual "Verify full chain" button

### Task 5: OpenInference Export — Item 12

**Files:**
- Create: `shell/src/runtime/openinference-export.ts`
- Create: `shell/src/runtime/__tests__/openinference-export.test.ts`
- Create: `docs/decisions/openinference-mapping.md`

**Decision doc:** Maps our entry fields to OI span attributes. Documents why we keep our custom schema (audit/integrity) and use OI as an export layer (observability).

**Exporter:** `exportToOpenInference(entries)` → OI-compatible spans:
- `span.name` = action (tool name)
- `span.kind` = "CHAIN" for commits, "TOOL" for reads
- `span.attributes` = { caller_class, identity, mode, params, provenance (custom) }
- `span.start_time` / `span.end_time` from timestampMs
